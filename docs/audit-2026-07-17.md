## Findings

### 1. `peace_league_website/utils/seed_data.py` — passthrough orchestrator (NOT APPLIED)

Tag: `shrink:` (borderline `yagni:`)
What: Six `from … import …` re-exports at module top, then `generate_test_data()` orchestrates them in order.
Replacement options considered:
- Inline `api.py` to import directly from `utils/seed/` and delete `utils/seed_data.py`.
- Move `generate_test_data()` to `utils/seed/__init__.py` and delete the seam file.

Risk: multi-file refactor (api.py + seed_data.py + __init__.py). The existing `api.py::seed_causes(force=0)` endpoint already does real type coercion (`bool(int(force))`) for HTTP string params, so the indirection is not pure noise. The file's own docstring marks it `# kept for backward compatibility`.
Decision: do not apply this run. Ship the audit only.

### 2. `peace_league_website/api.py::generate_test_data()` and `::seed_causes(force=0)` — HTTP-bound wrappers (NOT APPLIED)

Tag: `shrink:`
What (corrected): The two endpoints in `api.py` look like pure passthroughs but each carries two real reasons to exist:
1. `@frappe.whitelist(allow_guest=True)` decoration registers them at `POST /api/method/peace_league_website.api.<name>`. Removing the wrapper breaks the HTTP exposure path; the underlying `utils.seed_data.*` functions are NOT whitelisted on their own.
2. The wrappers normalize the public contract: `seed_causes(force=0)` accepts a stringifiable HTTP param and coerces with `bool(int(force))`; `generate_test_data()` carries the public docstring contract.

Replacement considered: move the `@frappe.whitelist` decorator onto `utils.seed_data.generate_test_data()` itself (and `utils.seed.seed_causes.seed_causes`) and either delete the `api.py` re-bindings or keep them as thin aliases.

Risk:
- The HTTP path `peace_league_website.api.generate_test_data` is part of the public contract. Any external caller (CI fixtures, smoke scripts, `scripts/smoke-bench.sh`) hitting that path breaks unless an alias is left in place.
- Coercion in `api.py::seed_causes(force=0)` may diverge from how `utils.seed.seed_causes.seed_causes` validates input (the wrapper is currently the seam).
- One file in `frappe-bench/scripts/` references `generate_test_data` per the smoke test convention; renaming the path requires updating that caller.

Decision: do not apply this run. Requires: (a) a list of consumers of the current HTTP paths, (b) decision on aliasing, (c) coordination with `scripts/smoke-bench.sh`.
