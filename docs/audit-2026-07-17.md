# Ponytail Audit — 2026-07-17

One-shot, read-only scan of the `peace_league_website` backend tree following the AGENTS.md contract. Following `ponytail-audit`: lists findings, applies nothing. `One fix per run` per `loop-constraints.md` — no fix landed in this run because no candidate cleared the safety bar (multi-file refactor risk > lazy savings).

## Method

Grep for the canonical `ponytail-audit` proxies:

- Single-implementation interfaces, abstract classes with one concrete
- Factories with one product, passthrough wrappers with one external caller
- Hand-rolled stdlib (custom validators, custom datetime, custom email parsing)
- Files exporting one thing, dead flags, hand-rolled caches

## Findings

### 1. `peace_league_website/utils/seed_data.py` — passthrough orchestrator (NOT APPLIED)

Tag: `shrink:` (borderline `yagni:`)
What: Six `from … import …` re-exports at module top, then `generate_test_data()` orchestrates them in order.
Replacement options considered:
- Inline `api.py` to import directly from `utils/seed/` and delete `utils/seed_data.py`.
- Move `generate_test_data()` to `utils/seed/__init__.py` and delete the seam file.

Risk: multi-file refactor (api.py + seed_data.py + __init__.py). The existing `api.py::seed_causes(force=0)` endpoint already does real type coercion (`bool(int(force))`) for HTTP string params, so the indirection is not pure noise. The file's own docstring marks it `# kept for backward compatibility`.
Decision: do not apply this run. Ship the audit only.

### 2. `peace_league_website/api.py::generate_test_data()` — docstring-only wrapper (NOT APPLIED)

Tag: `shrink:`
What: Whitelisted endpoint, body is `return _generate_test_data()`. The wrapper contributes no logic beyond a docstring comment block.
Replacement: import directly as `generate_test_data` from `utils/seed_data.py` (or from its new home after finding 1 is applied) and let Frappe's `@frappe.whitelist` decorate the underlying function.
Risk: changes the whitelisted function name (currently `peace_league_website.api.generate_test_data`). Any external HTTP caller using that path needs to know. Webhook integrations may break.
Decision: do not apply this run. Requires a heads-up to API consumers first.
