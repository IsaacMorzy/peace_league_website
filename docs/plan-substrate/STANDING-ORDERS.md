# Standing Orders — from plan-repo canonical

> Source: IsaacMorzy/peace-league-website-plan/AGENTS.md § Standing orders. **CANONICAL** on the plan-repo. THIS file is a **read-only mirror** for the bench agent — the plan-repo wins on conflict. Edit the canonical file in the plan repo, re-mirror here.

1. **Nginx on :443 is sacred.** Never enable Tailscale Funnel on this host; tailscaled grabbing `100.68.53.88:443` collides with the apex Let's Encrypt cert. Tailnet-only HTTPS MUST be on a non-443 high port (current state: `:8443` via `tailscale serve --bg --https=8443 http://127.0.0.1:80`). See `scripts/TAILSCALE-NOTE.md`.
2. **No plaintext sudo passwords in any script.** Use scoped `NOPASSWD` rules in `/etc/sudoers.d/peace-league-<feature>`. The rule MUST match the exact command string with no wildcards that could let the script overwrite `/etc/shadow` or similar.
3. **No new `peaceleagueafrica.com` URLs in Astro source or scripts.** Canonical / OG / Twitter / JSON-LD all use `peaceleagueafrica.org`. Email addresses too.
4. **All nginx config edits go via `scripts/*-nginx` -> `/etc/nginx/conf.d/` via `deploy.sh --prod`.** Direct `sudo cp` outside the sudoers rule is forbidden.
5. **Astro marketing pages live at `sites/peaceleagueafrica.localhost/public/`.** Don't write HTML into `apps/.../peace_league_website/peace_league_website/public/astro_pages/` unless you ALSO copy it into the site's `public/`; nginx try_files resolves against the latter.
6. **One ticket per session.** No batching. Spec -> implement -> gate -> sync for ONE ticket, then stop.
7. **Pre-conditions for opening any `wayfinder:task` ticket.** Stage 4 invokes `sudo nginx -t` and `sudo visudo -c -f /etc/sudoers.d/peace-league-deploy`. Both are wired through the scoped NOPASSWD rule at `/etc/sudoers.d/peace-league-deploy` (mode `0440 root:root`, four-command allowlist). Before claiming a `wayfinder:task` ticket the agent MUST `stat -c "%a %U:%G" /etc/sudoers.d/peace-league-deploy` and confirm `0440 root:root`. If the rule is missing or owned by a non-root user: **claim local ordinal `02` (GitHub issue `#3`) first** per `MAP.md`'s Ticket -> GitHub map. Ticket #2 is precisely the bootstrap that adds `sudo -n true` to `deploy.sh --prod`, after which subsequent `wayfinder:task` tickets can claim without manual sudo intervention.
8. **Follow DESIGN.md for every frontend implementation.** The Mintlify-derived design system locks: Inter (prose) + Geist Mono (code); `{colors.brand-green}` (`#00d4a4`) accent-only; `{colors.primary}` (`#0a0a0a`) on light, `#ffffff` on dark hero; `{rounded.full}` (pill) on buttons, `{rounded.lg}` (12px) on cards, `{rounded.md}` (8px) on inputs/code-blocks; `{typography.body-md}` (16px / 1.50 line-height) for prose. After any structural edit to DESIGN.md itself, run `npx @google/design.md lint DESIGN.md` from `$BENCH`.
