# Architecture Decision Records

One file per non-trivial decision. Filename: `NNNN-short-slug.md` where NNNN is a zero-padded sequence. Status flow: `proposed` → `accepted` → `superseded by NNNN` or `rejected`.

## Template

```markdown
# NNNN. <Title>

- Status: proposed | accepted | superseded | rejected
- Date: YYYY-MM-DD
- Deciders: <who>

## Context
<one paragraph: the pressure that forced the choice>

## Decision
<the call that was made>

## Consequences
<trade-offs, what becomes easier, what becomes harder>

## Alternatives considered
<one line each>
```
